# Glossary

A reference list of technical terms used throughout this guide.

| Term | Definition |
|------|------------|
| **API** (Application Programming Interface) | A set of rules that allows software applications to communicate with each other. |
| **Collection** | A folder in Postman used to group, organize, and share related API requests. |
| **Collection Variable** | A variable scoped to a specific collection, accessible by all requests within that collection. |
| **Collection Runner** | A Postman feature that executes all requests in a collection sequentially and displays the results. |
| **CRUD** | An acronym for the four basic data operations: **C**reate, **R**ead, **U**pdate, and **D**elete. |
| **DELETE** | An HTTP method used to remove a resource from the server. |
| **Endpoint** | A specific URL that an API exposes to perform a particular operation (e.g., `/get`, `/post`). |
| **Environment** | A named set of variables in Postman that can be applied to requests to avoid hardcoding values like URLs or API keys. |
| **Environment Variable** | A reusable key-value pair stored in a Postman environment, referenced in requests using `{{variable_name}}` syntax. |
| **GET** | An HTTP method used to retrieve data from a server without modifying it. |
| **HTTP** (HyperText Transfer Protocol) | The protocol used to transfer data between a client and a server on the web. |
| **HTTP Request** | A message sent from a client to a server asking it to perform an action, such as retrieving or submitting data. |
| **HTTP Response** | The message a server sends back after receiving a request, including a status code and optional data. |
| **JSON** (JavaScript Object Notation) | A lightweight, human-readable data format commonly used to send and receive data in API requests and responses. |
| **POST** | An HTTP method used to send new data to a server, typically to create a resource. |
| **Postman Echo** | A public API provided by Postman for testing HTTP requests, which echoes back the data it receives. |
| **PUT** | An HTTP method used to replace an existing resource on the server with new data. |
| **Query Parameter** | A key-value pair appended to a URL after `?` to filter or customize a request (e.g., `?foo1=bar1`). |
| **Request Header** | Metadata included in an HTTP request that provides additional context, such as the data format or authentication token. |
| **Request Body** | Data sent along with a POST, PUT, or PATCH request, typically formatted as JSON. |
| **REST** (Representational State Transfer) | An architectural style for designing APIs that uses standard HTTP methods to access and manipulate resources. |
| **RESTful API** | An API that follows REST principles, using URLs to identify resources and HTTP methods to define actions. |
| **Response Body** | The data returned by the server in response to a request, often formatted as JSON. |
| **Status Code** | A three-digit number in an HTTP response indicating the result of the request (e.g., `200 OK`, `404 Not Found`). |
| **URL** (Uniform Resource Locator) | The address used to identify a resource on the web (e.g., `https://postman-echo.com/get`). |
